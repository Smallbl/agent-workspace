【任务编号】TASK-20260402-002
【任务标题】修复编辑保存后列表不刷新 + 服务器重启检测机制
【问题描述】
1. 用户修改任务日期后保存，任务没有从当前日期列表即时消失，仍显示在原日期
2. 代码更新后服务器没有自动检测需要重启

【已定位信息】
1. 保存后前端只调用了 renderTasks() 和 updateStats()，但没有重新从 API 获取数据
2. 当前 renderTasks() 依赖本地 tasks 数组，保存成功后虽然本地数组更新了，但筛选逻辑没有重新过滤

【需要修复内容】
1. 修复 daily.html：保存成功后，如果是日期变更，应该重新加载任务列表
   - 方案A：保存后重新调用 loadTasks()
   - 方案B：保存成功后如果日期变了，切换到新日期的视图
2. 建议：可考虑添加服务器文件监控，检测到 server.py 变更时提示重启

【预期结果】
- 编辑任务修改日期 -> 保存 -> 任务立即从当前列表消失（显示正确的筛选结果）
- server.py 更新后，下一次请求时自动检测并提示重启

【超时设置】10分钟
