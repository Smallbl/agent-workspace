## 五、工作流程机制

### 5.1 主控协调模式（豆浆）

```
王 → 豆浆（主控）
     ↓
  理解需求
     ↓
  分解任务
     ↓
  ┌─────────────────────────┐
  │ 派发给专业 Agent         │
  │ (前端官/后端官/测试官)   │
  └─────────────────────────┘
     ↓
  校验结果
     ↓
  豆浆统一回复王
```

**⚠️ 铁律：主Agent绝不自己写代码**

### 5.2 简报推送流程

```
Cron触发
  ↓
生成简报（搜索+整理）
  ↓
输出到 memory/cron-output/
  ↓
豆浆主动推送飞书
  ↓
王收到飞书消息
```

**问题：简报任务绑定在 main，但飞书_chat工具只能在main session用，需要确认 announce 机制**

### 5.3 记忆管理流程

```
每日 00:00 (memory-agent)
  ↓
记忆备份
  ↓
git commit + push → GitHub
  ↓
每周一 09:00 (memory-agent)
  ↓
周回顾 → 更新 MEMORY.md
```
