### 备援脚本返回0条（2026-04-17 → ✅ 已解决）
- 根因：后端Agent修改后引入了bug，同时 openclaw-fallback.sh 9:30 与 cron 9:00 重叠导致双重推送
- 修复：删除 crontab 中的 9:30 任务，保留 00:30/08:30/12:00/17:30 四个备援窗口

### OpenClaw日报版本错误（2026-04-20 → ✅ 已解决）
- 问题：cron prompt配置不完整，漏了版本获取步骤，导致子agent编造版本号
- 根因：prompt中没有要求执行 `openclaw --version` 获取真实版本
- 修复：更新cron prompt，第一步强制获取版本，每条新闻必须包含原文链接
- 教训：配置cron/agent prompt时必须要求获取实际状态，不能让agent自己编造信息

### cron + 备援脚本双重执行导致重复推送（2026-04-21 → ✅ 已解决）
- 现象：工作简报同一天推送了两次
- 根因：cron任务（ai-work-briefing）11:02执行并推送，12:00备援脚本再次执行并推送
- 分析：cron配置11:30但实际提前约28分钟执行属正常偏差，备援脚本12:00按计划再次执行
- 修复：删除9:30 crontab备援任务，避免与cron 9:00重叠
- 备注：备援脚本与cron时间窗口重叠时会有重复风险
